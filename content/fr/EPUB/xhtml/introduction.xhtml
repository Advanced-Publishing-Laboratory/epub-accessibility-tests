Introduction

Statut de ce document

Cette publication est actuellement considéré comme [stable] par le consortium DAISY et vient en soutient aux efforts du W3C EPUB 3 Community Group.

Cette publication fait partie d'un ensemble intitulé "EPUB 3 Accessibility Test Suite" (Suite de tests d'accessibilité pour EPUB 3).

À propos de ce document

Ce document contient des tests fondamentaux pour l'accessibilité des dispositifs de lecture pour les utilisateurs en situation de handicap. Les tests avancés sont contenus dans une publication EPUB séparée que l'on peut trouver dans la suite de tests.

Un dispositif doté d'un support fondamental de l'accessibilité doit permettre de lire grâce à la vue, à l'ouïe ou au toucher. Cela signifie qu'il doit être possible pour un utilisateur de lire le contenu de ce document:

  • En paramétrant l'affichage, par exemple en ajustant la taille de la police et la couleur des caractères
  • en lisant le texte grâce à un lecteur d'écran et à la fonctionnalité de lecture vocale intégrée au dispositif
  • en lisant le texte avec un afficheur braille

Conventions

Les conventions suivantes sont utilisées out au long de ce document:

Les tests sont nommés grâce à un identifiant unique (ID), comme "file-010". Ces identifiants correspondent aux entrées dans les formulaires d'évaluations disponibles sur www.epubtest.org

Réaliser les tests

Le dispositif de lecture sera évalué par différents testeurs en utilisant différentes combinaisons de technologies d'assistance. Un rapport de tests sera produit pour chaque combinaison. Les testeurs pourront réaliser les tests décrits dans ce livre en utilisant le clavier, la souris ou des gestes tactiles. En terme de restitution, les testeurs peuvent utiliser l'audio (via un lecteur d'écran et/ou la fonctionnalité de lecture sonore intégrée), des ajustements visuels ou un afficheur braille.

Si vous êtes un utilisateur non ou malvoyant se servant d'un lecteur d'écran, ou si vous testez un dispositif de lecture sans écran, vous aurez probablement à compléter les sections suivantes :

  • Tests des fonctionnalités de base
  • Tests de navigation
  • Mises en évidence par surlignements, annotations et signets
  • Tests de lecture non visuelle

Si vous êtes un testeur voyant, avec des troubles visuels ou si vous avez des troubles de l'apprentissage tels que la dyslexie, vous aurez probablement à compléter les sections suivantes :

  • Tests des fonctionnalités de base
  • Tests de navigation
  • Mises en évidence par surlignements, annotations et signets
  • Tests des ajustements visuels
  • Tests de la fonctionnalité de lecture audio

Chaque test inclut une description des objectifs visés suivie par la définition du test à réaliser, qui peut être évaluée par vrai ou faux.

Le résultat de chaque test doit être enregistré dans le champ correspondant du formulaire sur le site Web. Les commentaires sont très utiles et doivent être renseignés en utilisant l'espace prévu à cette effet dans le formulaire.

Les tests décrits dans ce livre soient conçus pour être explicites, les évaluateurs et les développeurs peuvent juger utile de se référer aux orientations supplémentaires qui peuvent être téléchargées sur www.epubtest.org/instructions
